## TL;DR（チームの一致結論）

**「インストール不要・URL1本で即対戦できる “爆弾.io の決定版”」** として、空白の爆弾.io
ジャンルに独自ブランドで一番乗りする。
収益は **ポータル＝無料の母数製造機（広告）／課金エンジンはポータル外（Steam無料版＋スキン）**
の二層構造。送客はゲーム内リンク（規約違反）に頼らず **ブランド名の指名検索** で引く。
差別化は欲張らず **「爆弾を蹴る」1機能（3.5人日）** だけ初回に入れて埋没を回避する。

## 合意した4論点

### Q1. ブランド名 → リネーム（全員一致）

- 「Bom Bom Panic」は .io 層に刺さらず検索でも埋没。元ネタ感も強い。
- 短く・口に出せる・被らない独自名へ（例: Boomies 系）。
- **命名の必須3条件**（候補出しの段階でセットで確認）:
  1. 検索でユニークに勝てる（既存ゲーム/一般語と被らない）
  2. 商標取得可
  3. ドメイン空き
- **理由**: 送客（Q2）がブランド名検索頼みなので、名前の独自性＝収益の生命線。
- **技術コストは低**（ドメイン/パッケージ名/Colyseus ルーム名のみ）。「Bomberman」類似は
  商標・ストア審査リスク → **課金基盤を作る前の“今”やる**のがベスト。

### Q2. ローンチ主戦場とマネタイズ → 二層ハイブリッドの段階移行（全員一致）

| 層 | 役割 | 手段 |
|---|---|---|
| **集客（入口）** | 無料の母数製造機・リテンション検証 | CrazyGames / Poki / itch.io。広告レベシェアで回収、**LTVは積まない割り切り** |
| **収益（本拠地）** | 課金エンジン | **Steam 無料版＋スキン課金** / itch / 自社サイト。Stripe は自社/Steam前提 |

- **技術的事実**: ポータルは広告収益のみで、独自アカウント＋Stripe課金は構造的に載らない。
  → 課金するなら自社サイトか Steam が前提。
- **送客はプル型**: ゲーム内の直リンク送客（＝ポータル規約違反）に頼らず、
  ブランド名の指名検索／縦型クリップの概要欄／Discord で外部へ引く。
- **譲れない点**:
  - ポータルの**独占条項は飲まない**（自社/Steam の別ビルド保持権を残す）。
  - リネーム名は**ドメイン＋商標を自社確保してから**使う。
- **順序**: 同時両立ではなく段階移行。広告で母数 → その実績が B2B 営業の弾にも課金の前提にもなる。

### Q3. ローンチ前に作る新メカニクス量 → 「最小改修 ＋ 差別化1機能」（決着）

- 無改修の劣化コピーは即埋没する。だが差別化は欲張らない。
- **MVP の差別化メカ＝「爆弾を蹴る」1個のみ**。
  - グリッド方式（マス単位移動、Matter 動的ボディ不要 / 爆弾 state に movingDir + targetTile）
  - 等速直線・跳ね返りなし／壁・箱・プレイヤーで停止・押し込みなし
  - **移動入力での重なりで自動発火（新規ボタン不要＝モバイル互換）**
  - **蹴られた爆弾は到達マス確定後にタイマー再評価**（移動中に手元で爆発する“理不尽死”を防ぐ保険＝リテンション直結。因果が読めて演出的にも正しい）
  - 実装 **5人日（確定・安全側）**。グリッド方式で巻ければ前倒しのアップサイドあり。
    工数の変遷: 3.5（フル最小ケース）→ 6（フル物理前提）→ **5（A/B/C簡素化＋タイマー再評価保険込み）**。
    **課金基盤と完全並行で実装可**。
- **bot 蹴りは難易度連動**（使用頻度 0.1〜0.7、初回戦は弱 bot がデモ的に蹴って見せる）。
- **「投げる」「ゴースト介入」はローンチ後 / シーズン2の目玉に温存**。
- 課金は**コスメのみ（Pay-to-Win 回避）**。

### Q4. 最初の90日クリティカルパス（統合）

3レーンを依存関係順に統合した実行計画。**蹴る実装は技術1〜3と並行**。

```
Phase 0（基盤・最初にやる / 低コスト）
  └ リネーム（ドメイン＋商標確保 → 名称変更）
  └ Docker スリム化 1.2GB → 300MB（standalone + alpine + 本番deps）

Phase 1（公開できる最小版を作る）
  ├ 製品: Quick Brawl ＋ 摩擦ゼロのリマッチ（3秒で「あと1回」）← リテンションの心臓
  ├ 製品: 「爆弾を蹴る」実装（5人日・確定 / 課金基盤と並行）← 埋没回避の最小投資
  └ 製品: bot 戦オンボーディング（初回必勝＋蹴りをデモで見せる）

Phase 2（母数を取る・検証する）
  ├ マーケ: ポータル審査通過 → 公開
  ├ マーケ: 縦型クリップ毎日投下＋招待URL＋Discord（バイラル＆過疎対策）
  └ 計測: retention（D1/D7）・同接・バイラル係数K を測定し勝ち筋を特定

Phase 3（課金を載せる ※数字が立ってから）
  ├ 技術: Supabase 認証＋Postgres → スキーマ（プロフィール/インベントリ/購入履歴）
  ├ 技術: Stripe Checkout＋Webhook（サーバーで付与・クライアント結果は不信）
  ├ 技術: Sentry＋構造化ログ＋入力レート制限/Webhook署名/monitor露出対策
  └ 製品/事業: Steam 無料版＋スキン課金導線、B2B 営業、デイリー/シーズンパス

後回し（同接が詰まってから / シーズン2）
  └ スケール: Colyseus 0.15 移行 / Redis Presence / 多リージョン(Fly.io)
  └ 製品: 投げる・ゴースト介入・カスタムマップ(UGC)
```

## KPI（初期に追う順）

1. **D1 / D7 リテンション**（短セッション対戦の生命線。目安 D1 30%+ / D7 10%+）
2. **DAU とバイラル係数 K**（招待→流入。K>0.3 で広告費ほぼゼロ成長）
3. **広告 ARPDAU**（ポータルでまず実測）
4. **指名検索ボリューム**（ブランド名 = 送客導線の太さ）
5. **課金率 / ARPPU**（Steam/自社で。payer 0.5〜2% を初期目標）

## 価格感（仮説）

- 単品スキン: ¥150〜¥600 / バンドル・シーズンパス: ¥600〜¥1,200
- 広告除去（買い切り）: ¥980 前後
- B2B プライベートルーム（イベント1回）: ¥30,000〜¥100,000

## インフラ費用感（オーダー）

- DAU 100: $5〜20/月（Railway か Hetzner 1台 + 各無料枠）
- DAU 1,000: $50〜150/月
- DAU 10,000: $500〜2,000/月（帯域と Postgres が主因）
